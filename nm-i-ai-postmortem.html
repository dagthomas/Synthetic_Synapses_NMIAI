Architecture

Two-Stage Ensemble Pipeline

Input 1536px → YOLOv8x-A (TTA) + YOLOv8x-B (TTA) → WBF Fusion → Crop 256px → ConvNeXt-V2 → Blend + Filter

Detection (70% of score)

  • Dual YOLOv8x models, different seeds
  • Weighted Boxes Fusion merges predictions
  • Cloud-trained on A100 GPUs
  • Confidence threshold: 0.001 (max recall)

Classification (30% of score)

  • ConvNeXt-V2 nano (15M params, 30MB)
  • Class 356: "background" false-positive filter
  • Score blending on agreement/disagreement
  • 24 submission iterations to dial in
06
War Stories

Things That Went Sideways

  • Missing .jpeg files — glob used *.jpg, missed 38 images. Trained on 210/248 images without noticing
  • SAHI tiling backfire — expected to help small products, actually created massive conflicting detections (-0.009 mAP)
  • Rotation augmentation disaster — degrees=10.0 on upright grocery products. Products don't rotate IRL
  • 1920px inference on 1280px-trained model — higher res made it worse (-0.011 mAP)
  • Mixup augmentation creates ghost products on dense shelves
  • Background rejection class (356) — trained on empty shelf crops, filters false positives for free
07
Key Insight
Train set mAP saturated at 0.978 across all configs. The 0.07 gap to leader wasn't model quality — it was generalization to unseen shelves.

The Black Hole

29 Tries. 0 Points.

The invoicing pipeline consumed our soul and returned nothing.

  • Tasks 10, 11, 12, 16: all invoice variants
  • 29 attempts across all four
  • Missing invoiceDueDate? Wrong orderLines JSON?
  • The agent architecture works fine for Task 17 (3.5/4.0)
  • The bug was in the tools, not the agent

Lesson learned:

When a task type fails 29 times, debug the tool directly against the API. Don't throw more agent turns at it.

Also: accounting is inherently adversarial.
